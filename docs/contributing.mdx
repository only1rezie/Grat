We welcome contributions to all parts of the Prism ecosystem!

## Areas for Contribution

### 1. Error Taxonomy
The most impactful way to help. Adding plain-English explanations for cryptic errors helps every developer.

### 2. Core Engine (Rust)
Improving state reconstruction speed, adding new trace annotations, or optimizing the replay sandbox.

### 3. Web & IDE Interfaces
Enhancing the visual trace viewer or adding new VS Code Quick Fixes.

## Development Setup

Follow the [Installation from Source](/installation) guide to set up your local environment.

## Pull Request Process

1. Fork the repository and create your feature branch.
2. Ensure all tests pass (`cargo test`, `pnpm test`).
3. Update documentation if you add new features or CLI flags.
4. Submit your PR and wait for a maintainer review.
